# SWP - Hausübungen und Unterrichtsstoff

Dieses Repository enthält meine Hausübungen sowie den Stoff, den wir im Unterricht der Softwareentwicklung (SWP) durchnehmen. Die Projekte sind nach Datum organisiert und decken verschiedene Themen der C#-Programmierung ab, von grundlegenden Konsolenanwendungen bis hin zu Web-Applikationen und KI-Inferenz.

## Projekte

### 2025-09-08
Einführungsprojekt zur C#-Programmierung.

### 2026-03-11_Inventarsystem
Eine Konsolenanwendung, die ein Inventarsystem mit Gegenständen wie Waffen und Heiltränken simuliert. Verwendet Interfaces und Klassen zur Objektorientierung.

### 2026-03-18_Einkaufsliste
Eine Einkaufslisten-Anwendung, die das Hinzufügen, Überprüfen und Vergleichen von Artikeln ermöglicht. Demonstriert Listen-Handling und String-Vergleiche.

### 2026-03-25_Lagerverwaltung
Ein Lagerverwaltungssystem zur Verwaltung von Produkten und Beständen.

### 2026-04-08_Inference
Einführung in KI-Inferenz: Erklärung von Inference, kostenlosen und günstigen API-Optionen (z.B. Groq, Google AI Studio, OpenRouter).

### 2026-04-15_Blazor
Eine Blazor Server-Anwendung für interaktive Weboberflächen.

### 2026-04-22_Inference
Weiterführende Themen zu KI-Inferenz.

### 2026-04-22_Traumapp
DreamDay - Eine Web-Applikation zur Tagesplanung, die Wetterdaten (Open-Meteo), Musik (Spotify) und Rezepte integriert.

## Voraussetzungen

- .NET 8.0 SDK (für C#-Projekte)
- Für Web-Projekte: Ein Webbrowser
- Für KI-Projekte: API-Keys von den entsprechenden Diensten (siehe Projekt-READMEs)

## Installation und Ausführung

1. Klone das Repository:
   ```
   git clone <repository-url>
   cd SWP
   ```

2. Für jedes C#-Projekt:
   - Navigiere in den Projektordner (z.B. `cd 2026-03-11_Inventarsystem`)
   - Baue das Projekt: `dotnet build`
   - Führe es aus: `dotnet run`

3. Für Blazor-Projekte:
   - Navigiere in den Projektordner (z.B. `cd 2026-04-15_Blazor/BlazorApp1`)
   - Baue und führe aus: `dotnet run`
   - Öffne den Browser unter der angegebenen URL (normalerweise `https://localhost:5001`)

4. Für Inference-Projekte:
   - Lies die entsprechenden README-Dateien für API-Setup und Nutzung.

## Struktur

Das Repository ist chronologisch nach Unterrichtsdaten organisiert. Jeder Ordner enthält ein separates Projekt mit eigenem `Program.cs` und ggf. zusätzlichen Dateien wie `README.md` oder Konfigurationsdateien.
